## 1. Problem Statement

Today LexCursor is three standalone HTML files served by `python -m http.server`,
with no framework, build step, routing, shared components, or place to add a
backend or auth. Nothing real can be built on top of static files, so every
downstream feature is blocked. Scaffolding a Next.js app converts the mockups
into a maintainable, deployable foundation and unblocks all subsequent stories.

## 2. Goals

- Stand up a Next.js (App Router) + TypeScript + Tailwind project that builds and
  runs locally and in CI.
- Port the 3 mockups to real routes (`/dashboard`, `/editor`, `/linter`) with a
  shared layout + sidebar component.
- Encode the existing design tokens from `DESIGN.md` into the Tailwind config.
- Establish project conventions (lint, format, path aliases, folder structure)
  that later stories follow.

## 3. Non-Goals

- No authentication (F1.2), database, or live data (F1.3, C2.1) ‚Äî screens render
  static/sample content for now.
- No AI features, document parsing, or compliance logic.
- No Docker or CI/CD (F1.5, F1.6) beyond a basic build check.

## 4. Personas & User Stories

- **As a developer**, I want a single `npm run dev` to launch the whole app so I
  can build features without juggling static files.
- **As an associate attorney (demo)**, I want to navigate between Dashboard,
  Editor, and Linter via the sidebar so the product feels like one application.
- **As a designer**, I want the design tokens centralized so UI stays on-brand.

## 5. Functional Requirements

- **FR-1.** The app MUST be a Next.js App Router project in TypeScript with
  Tailwind CSS configured.
- **FR-2.** The app MUST expose routes `/` (redirect or landing), `/dashboard`,
  `/editor`, `/linter`, each rendering the ported mockup content.
- **FR-3.** A shared `AppLayout` MUST render the sidebar + top bar; the sidebar
  MUST link between the three screens and highlight the active route.
- **FR-4.** Tailwind config MUST include the color, typography, spacing, and
  radius tokens defined in `lexcursor_visual_language/DESIGN.md`.
- **FR-5.** The project MUST include lint (ESLint) and format (Prettier) configs,
  and `npm run build` MUST succeed with zero type errors.
- **FR-6.** The repo SHOULD define path aliases (e.g. `@/components`, `@/lib`).
- **FR-7.** A `README.md` MUST document install, dev, build, and folder layout.

## 6. Non-Functional Requirements

- **Performance** ‚Äî dev server cold start < 5s on a typical laptop; production
  build of the 3 routes < 30s in CI.
- **Security** ‚Äî no secrets committed; `.env*` gitignored; dependencies installed
  from lockfile.
- **Privacy & Compliance** ‚Äî N/A ‚Äî no user data handled at this stage.
- **Accessibility** ‚Äî ported screens MUST preserve semantic landmarks
  (`<nav>`, `<main>`, `<header>`) and reach WCAG 2.1 AA for color contrast.
- **Scalability** ‚Äî folder structure MUST support feature-based growth (route
  groups, `components/`, `lib/`, `server/`).
- **Reliability** ‚Äî `npm run build` is the gate; a broken build blocks merge.
- **Observability** ‚Äî N/A ‚Äî added in P5.2.
- **Maintainability** ‚Äî ESLint + Prettier enforced; strict TypeScript on.
- **Internationalization** ‚Äî copy SHOULD be centralized to ease later i18n; full
  i18n is out of scope.
- **Backward compatibility** ‚Äî keep the original static mockups in-repo (read-only
  reference) until parity is verified, then remove in a follow-up.

## 7. Acceptance Criteria

- **AC-1.** *Given* a clean clone, *When* a dev runs `npm install && npm run dev`,
  *Then* the app serves at `localhost:3000` with no console errors.
- **AC-2.** *Given* the running app, *When* the user clicks each sidebar item,
  *Then* the route changes to `/dashboard`, `/editor`, `/linter` and the active
  item is visually highlighted.
- **AC-3.** *Given* the codebase, *When* CI runs `npm run build` and `npm run lint`,
  *Then* both pass with zero errors.
- **AC-4.** *Given* any ported screen, *When* it renders, *Then* it visually
  matches the original mockup within reasonable tolerance using DESIGN.md tokens.

## 10. UI / UX

- **New routes:** `/dashboard`, `/editor`, `/linter`; `/` redirects to
  `/dashboard`.
- **New components:** `AppLayout`, `Sidebar`, `TopBar`, `NavItem`.
- **Flows:** (1) land ‚Üí redirected to dashboard; (2) sidebar navigation between
  the three screens.
- **States:** static content only; ensure a sensible 404 page exists.
- **Responsive:** sidebar collapses to a drawer < `md`, per DESIGN.md reflow.
- **Accessibility:** sidebar is a `<nav>` with `aria-current="page"` on the active
  link; logical focus order; visible focus rings.
- **Tokens:** all colors/spacing/typography come from the Tailwind token config,
  not hardcoded hex.
